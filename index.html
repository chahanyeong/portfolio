PROJECT 02

앱 헬스케어 시스템 구축

이슈 영향 시간 -94.47% · 이슈간 평균 약 30시간 감소

Dagster + Bitbucket으로 기존 서버를 보완하는
DevOps 인프라 개발

기존 운영의 한계

  • 주피터 노트북 직접 사용 → 버전 관리 불가
  • 크론탭 UI·로그 부재 → 점검 비용 큼
  • 라이브 서버 직접 수정·배포

현재 운영 인프라

  • Bitbucket 코드 버전 관리·롤백 체계 표준화
  • Bitbucket Runner CI/CD 자동 배포
  • Dagster UI로 실행 로그·작업 흐름 시각화
  • 크론탭 기반 얼럿 42개 → Dagster 마이그레이션 완료

기존 얼럿 시스템의 불확실성을 측정가능한 체계로 재구성

  • 자사 앱 운영 얼럿 44개 리뷰, 평가에 쓸 2025년 개발 이슈 History 75개 리스트 업
  • 평가 가능한 얼럿을 이슈 Recall · 정밀도 · 반응성 · 오탐률 4가지 지표로 정량 평가 + 개선점 도출
  • KPI 기반 반기 단위 평가·개선 사이클로 운영되는 얼럿 품질 관리 체계 수립

AI Engineering Practice

AI 에이전트로 일하는 방식

스케일 · 검증 · 도구 결합 · 학습.

01 · SCALE

에이전트 팀 활용

  • 에이전트 팀으로 기존 개발 이슈 120건의 정의·평가 수행
  • 이 과정을 워크플로우로 구조화 — 이후 이슈 발생 시 에이전트가 자동으로 평가·적재
02 · CROSS-CHECK

GPT · Claude
교차 검토 체계

  • GPT · Claude를 병행 활용한 교차 검토 체계 적용
  • 단일 모델 의존으로 인한 사고 편향을 줄이기 위해 GPT · Claude 모델 혼용
  • 두 모델 결과가 일치하면 신뢰, 불일치 시 사람이 검토 → 리뷰 비용 최소화
03 · TOOLING

플러그인 · 스킬 활용

  • superpowers 플러그인을 기저 하네스로 활용해 작업 품질 안정화
  • 예시: 자동 일정 캘린더 — Google Calendar에 운영 이벤트 자동 등록
04 · LEARNING

분석 데이터 자산화

  • 분석 결과물을 자산화 — 같은 분석이 필요할 때 에이전트에게 할당
  • 산출 결과물 공유 (예시: Mobile CCU 이상치 분석)
