PROJECT 01

매출 KPI · 지표 모니터링 자동화

2025년 하반기 매출 증감 예측 86%

주단위 매출 예측 → 주간 KPI 정의 → 4주 변화율 스코어링 자동화 → 이상치 발견 시 알럿 → 의사결정까지 이어진 파이프라인.

문제 정의

배포·이벤트 영향도를 정량 평가할 기준이 없었음.

접근

주단위 매출 예측을 KPI 임계선으로, 4주 평균 대비 변화율을 스코어화.

액션 트리거

변화율 ±N% 이상 또는 KPI 임계선 이탈 시 알럿 + 원인 후보 제언.

CASE A · 2026-04-09 확정

고액 상품 환산을
10% 인상했더니
매출이 +7.9%

  1. 01
    문제 발견
    KPI 보고에서 고액 상품 매력도가 떨어지는 신호 감지.
  2. 02
    데이터로 원인 짚음
    $499~$999 구간에서 배팅 금액이 비선형 급증 → 고액 결제자 환산 비율이 박했음.
  3. 03
    액션 결정
    $99 이상 고액 상품의 환산 코인 비율을 약 10% 인상해서 offering을 바로잡기로 결정. 저액 구간은 유지.
  4. 04
    결과
    적용 후 m/m 매출 +7.9% 증가, 고액상품 구매 비율 유지.
VERDICT m/m 매출 +7.9%, 고액상품 비율 유지.

PROJECT 02

앱 헬스케어 시스템 구축

이슈 영향 시간 40% 단축

Dagster + Bitbucket으로 기존 서버를 보완하는 DevOps 인프라 개발

기존 운영의 한계

  • 주피터 노트북 직접 사용 → 버전 관리 불가
  • 크론탭 UI·로그 부재 → 점검 비용 큼
  • 라이브 서버 직접 수정·배포

현재 운영 인프라

  • Bitbucket 코드 버전 관리·롤백 체계 표준화
  • Bitbucket Runner CI/CD 자동 배포
  • Dagster UI로 실행 로그·작업 흐름 시각화
  • 크론탭 기반 얼럿 42개 → Dagster 마이그레이션 완료

기존 얼럿 시스템의 불확실성을 측정가능한 체계로 재구성

  • 자사 앱 운영 얼럿 44개 리뷰, 평가에 쓸 2025년 개발 이슈 History 75개 리스트 업
  • 평가 가능한 얼럿을 이슈 Recall · 정밀도 · 반응성 · 오탐률 4가지 지표로 정량 평가 + 개선점 도출
  • KPI 기반 반기 단위 평가·개선 사이클로 운영되는 얼럿 품질 관리 체계 수립
