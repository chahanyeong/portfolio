PROJECT 01

KPI · 매출 예측 자동화

주단위 매출 예측 → 주간 KPI 정의 → 4주 변화율 스코어링 자동화 → 이상치 발견 시 액션 제언 → 실제 의사결정까지 이어진 파이프라인.

문제 정의

배포·이벤트 영향도를 정량 평가할 기준이 없었음.

접근

주단위 매출 예측을 KPI 임계선으로 사용, 4주 평균 대비 변화율을 스코어화.

액션 트리거

변화율 ±N% 이상 또는 KPI 임계선 이탈 시 알럿 + 원인 후보 제언.

의사결정 사례 — 데이터로 가격 정책을 바꿔 매출을 끌어올림

이 PROJECT의 핵심은 "지표를 본다"가 아니라 "지표에서 발견한 신호를 실제 액션으로 옮긴다".

CASE A · 2026-04-09 확정

VVIP 가격대 정책 리뉴얼

  1. 1 · 문제 인지

    인플레이션 영향으로 고액 결제 상품의 상대적 매력도(요율 후함)가 떨어지고 있음을 KPI 보고에서 감지.

  2. 2 · 데이터 분석

    "결제 코인 대비 배팅 금액(최빈)" 차트로 결제 가격대별 유저 행동 패턴 분석.

  3. 3 · 시뮬레이션

    가격대별 BaseCoin / 판매가격 % 요율 변경안 작성.

  4. 4 · 사전 검증

    상위상품 요율 증가가 게임 내 인플레이션(코인 가치 하락)을 야기하지 않는지 시뮬레이션.

  5. 5 · 액션 → 결과

    2026-04-09 변경안 적용. 인플레이션 이후 m/m 매출 +7.9% 증가, 고액상품 건수 구매 비율 유지.
