PROJECT 02

앱 헬스케어 시스템 구축

이슈 영향 시간 -94.47% · 이슈간 평균 약 30시간 감소

WHY

크론탭 + 주피터 임시 운영 · 얼럿 신뢰도를 측정할 KPI 부재

HOW

Dagster + Bitbucket DevOps 인프라 + 얼럿 4지표 정량 평가 체계

RESULT

이슈 영향 시간 -94.47% (평균 30시간 단축), 반기 평가·개선 사이클로 운영

Dagster + Bitbucket으로 기존 서버를 보완하는
DevOps 인프라 개발

  • 주피터 직접 사용 → Bitbucket 버전 관리 · 롤백 표준화
  • 크론탭 직접 등록 → Bitbucket Runner CI/CD 자동 배포
  • 라이브 서버 직접 수정 → Dagster Dev 검증 → 관리자 승인 → Live
  • 크론탭 기반 얼럿 42개 → Dagster 마이그레이션 완료

기존 얼럿 시스템의 불확실성을 측정가능한 체계로 재구성

  • 자사 앱 운영 얼럿 44개 리뷰, 평가에 쓸 2025년 개발 이슈 History 75개 리스트 업
  • 평가 가능한 얼럿을 이슈 Recall · 정밀도 · 반응성 · 오탐률 4가지 지표로 정량 평가 + 개선점 도출
  • KPI 기반 반기 단위 평가 · 개선 사이클로 운영되는 얼럿 품질 관리 체계 수립
